Compiling Instructions

This document discusses issues related to compiling Q Light Controller from source files. It is assumed that the user is installing QLC with the default settings.

Directory structure

  • debian: Scripts for debian package creation
  • docs: QLC documentation files
  • etc: Desktop integration stuff
  • fixtureeditor: Fixture editor sources
  • fixtures: Fixture definition files
  • gfx: Graphics, icons...
  • inputdevices: Input device definition files
  • libs
    • common: Common helper classes
    • dmx4linuxout: DMX4Linux output plugin
    • ftdidmx: FTDI-chip-based output plugin
    • hidinput: Input plugin for Human Interface Devices (mice, joysticks...)
    • llaout: Linux Lighting Architecture output plugin
    • midiinput: MIDI-based input plugin
    • midiout: MIDI-based output plugin
    • soundtolight: Deprecated (needs updating)
    • udmxout: Output plugin for Anyma uDMX devices
    • usbdmxout: Output plugin for Peperoni Rodin devices
  • main: The main QLC application sources

Configuration

In most cases, qlc doesn't require any manual configuration if you have all the necessary libraries installed on your system. The most common cases of unsuccessful compile operations are the result of missing or incorrect libraries. To compile QLC, you need at least these versions of the following components (the more recent the better):

  • Qt >= 4.1 development libraries & tools

Requirements - Linux
  • Main application, library and fixture editor: g++ >= 4.3
  • FTDIDMX Output plugin: No additional requirements
  • HID Input plugin: No additional requirements
  • MIDI Input plugin: libasound, libasound-dev, pkg-config
  • MIDI Output plugin: libasound, libasound-dev, pkg-config
  • LLA Output plugin: liblla, lla-dev, pkg-config (see libs/llaout/README)
  • uDMX Output plugin: libusb, libusb-dev
  • USBDMX Output plugin: No additional requirements

Requirements - Windows
  • FTDIDMX Output plugin: ?
  • HID Input plugin: Not available for Windows
  • MIDI Input plugin: No additional requirements
  • MIDI Output plugin: No additional requirements
  • LLA Output plugin: Not available for Windows
  • uDMX Output plugin: No additional requirements
  • USBDMX Output plugin: No additional requirements

Requirements - OSX
  • FTDIDMX Output plugin: No additional requirements
  • HID Input plugin: Not available for OSX
  • MIDI Input plugin: Not available for OSX
  • MIDI Output plugin: Not available for OSX
  • LLA Output plugin: liblla, lla-dev, pkg-config (see libs/llaout/README)
  • uDMX Output plugin: Not available for OSX
  • USBDMX Output plugin: Not available for OSX

Compiling & Installation

For windows, just install the latest Qt package from Trolltech/Nokia along with the default MinGW setup and start the Qt command prompt. For Linux or OSX, in addition to the Qt libraries, you need also Qt development packages.

As normal user, type to your X terminal/shell prompt/whatever:

        qmake

and then:

        make

and then (as administrator/root):

        make install

Package creation (linux/debian)

If you wish to create your own debian packages, there is a special debian directory for debian-specific package info. In general, you shouldn't have to modify anything in the directory, in fact, you don't need to go there at all. To be able to create debian packages, you'll need the following components, in addition to the ones needed to compile QLC:

  • debhelper
  • dpkg
  • dpkg-dev (namely dpkg-buildpackage)
  • fakeroot

To start building a debian package, just cd to the QLC top directory and type: 

	dpkg-buildpackage -rfakeroot

This way, if everything goes well, the scripts create two packages to the parent directory (cd ..):

  • qlc (The main application package)
  • qlc-fixtures (Additional fixture definitions)

Package installation (debian)

You can install the packages with:

	dpkg --install [package name].deb

And uninstall them with:

	dpkg -P [package name].deb
