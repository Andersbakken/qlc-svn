Coding Convention

This file describes the coding convention (ten commandments) used in QLC source code. The purpose of this document is to instruct developers to write somewhat uniform code. The following rules are more or less similar to the Qt coding convention. Please read that page first. It's REALLY good.

*/

2. Each member variable has a prefix "m_", for example:

	int m_fooMember;
Each global variable or function begins with "_", for example: 
	int _foo;
Still, try to avoid global variables at all costs!

3. Each function begins with a small letter and in compound words, all the following words begin with a capital letter. No word separation with underscore is allowed. Examples: 

int foo();
int fooBar();
int thisIsAMemberFunction();
int this_is_not_allowed_because_it_is_ugly;

4. Every control structure (if, while, for...) will be written inside brackets { } even if they would contain only one line of code, unless it looks really nice and clean. Also, brackets are written on their own line without anything else after or before the bracket on the same line. This goes for function and class definitions, too: 

do
{
    foo();
}
while (bar);

if (foo == true)
{
    foo();
}
else
{
    bar();
}

for (int i = 0; i != n; i++)
{
    foo(i);
}

void FooClass::fooBar()
{
    ...
}

5. Each class name will be written each word beginning with a capital letter. Same rules as with functions. 

class FooBar;
class Foo;

5.1. Respectively, each class should be in its own file and the file name must be the class' name, in small letters. If it makes things significantly more logical, SMALL classes can be defined inside other classes' definitions and files. File suffix for implementation files is ".cpp" and for header files ".h". Do NOT use other styles.
In general:
class FooBarClass definition is saved in foobarclass.h
and its implementation is in foobarclass.cpp

6. All macro definitions are written completely in capital letters, words may be separated with "_". Example: 

#define FOOBAR 5
#define FOO_BAR(x) foobar(x);

6.1 Normal integer macros, like the ones you would use to identify certain menu items, columns etc. can be substituted with static constants, in which case, the constant's name begins with a capital 'K'. This is done to associate them more into the field of #defined macros than variables.

static const int KMenuItem ( 1000 );

7. Each operator will be separated with spaces: 

if (foo == bar)

while (foo != bar)

int foo = bar;

for (int foo = 0; foo != bar; foo++)
...

8. Variable, function and class names should be descriptive. Use your common sense here.

9. Write nice and readable code. Do not write code in one pile, use spaces and empty lines between logical sections.

10. WRITE COMMENTS.
