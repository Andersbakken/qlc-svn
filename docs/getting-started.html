Starting QLC

  • Start qlc from you terminal by issuing the command qlc. You'll see some text on the terminal about version, plugins etc. QLC startup messages might look like:
    • 
--- Starting Q Light Controller 2 ---
This program is licensed under the terms of the GNU General Public License.
Copyright (c) 2000, 2001, 2002 Heikki Junnila (hjunnila@iki.fi)

Probing /usr/local/qlc/libs/ for plugin objects...
Library: Joystick | Version: 0.0.1
Found 1 plugin object(s)

    After these messages the main window will open. If there are any errors, refer to Errata and FAQ.

    Main Window

    Most controls throughout the software have tooltips, which you can see holding your mouse button over a control item (button, slider etc.) for a while. The menubar on the top of the workspace window contains vital operations which we will discuss a bit later. Below the menubar is the toolbar that contains the following buttons (from left to right): new workspace, open an existing workspace, save current workspace, open device manager and open virtual console.

    Settings

    The first thing you should do is to check that qlc's global settings are correct. Select Settings from File menu. The following dialog should appear:



    Now check:
  • The Pixmap Directory (depends on installation options, /usr/local/qlc/xpm by default)
  • The Device Directory (depends on installation options, /usr/local/qlc/devices by default)
  • UI Update Interval (not implemented)
  • Timer Frequency (should always be 1024)
  • Widget Style (you can slightly alter the look of qlc with this feature).
  • Visual Feedback (not implemented)

    • Click OK to accept and close the settings dialog.

    Device Manager

    The device manager is the heart of qlc's device-oriented architecture. As its name already says, you manage your input and output devices from the device manager. You can add or remove devices, edit and create new device classes (see glossary for the definition), assign DMX addresses and so on..

  • Open the device manager by pressing its button on the toolbar (fourth button from left).

    • Device Manager

    Here, the tool buttons are as follows: add new output device, add new input device, remove currently selected device, monitor selected device and view selected device's console.

  • Add an input device (press the first button from left) to our workspace. You should see a the following dialog:

    • Add New Input Device Dialog

    As you can see, a device class (Botex DPX-620) has already been selected in this example.

  • Now select Botex DPX-620.

    • You can edit the new device's name in the Name field and you can either set its address manually from the Address field or you can let QLC assign the next free contiguous address space. If you don't understand the dmx addressing principles, please consult your lighting equipment manuals for more info. Let QLC assign the address for now.

  • When you're done, press OK

    • Device Manager with DPX-620 as an output device

    In the left pane you can see the input and output devices that are present in your current workspace. In the right pane, you can see the currently selected device's information.
