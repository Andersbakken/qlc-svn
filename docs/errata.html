Problem Solution
I seem to be able to do nothing in QLC. How does this thing work? Open device manager, click add new output device and you're ready to rock'n'roll! :)
OK, I've added a device to device manager, but when I try to click the monitor or dmx console button, nothing happens. You must first select the device whose monitor or dmx console you want to view.
I've assigned a function to a button in virtual console but when I press the button, nothing happens or the function runs slooooowly. Create a speedslider to virtual console and assign a Speed Bus to it AND to the button. After this, dragging the speed slider will change the speed of your function.
I've built the analoque interface, but QLC cannot control any dmx channels. First, run the small script that checks for the necessary kernel modules. If the output on the console seems okay, but the problem persists, check that the parallel port is set to EPP in your BIOS configuration.
I got the message "Unable to access device directory: /foo/bar/blah". You don't have any fixture definitions in the directory. Either create some with dce or install qlc-fixtures package.
I got the message "Unable to open workspace: /foo/bar/blah.qlc". You have set an option in the settings dialog to try and open the last used workspace file on startup. Either create a new workspace file or clear the setting from settings dialog.
I got the message "Unable to save file". In most cases this means that you don't have write permission to the directory or the file (that already exists).
I got the message "Unable to access the plugin directory: /foo/bar/blah". The directory that QLC probes for plugins doesn't exist. You can use the program normally with the dummy plugin (no DMX hardware support). If you wish to have hardware support, we recommend installing the dmx4linux plugin (that should already be installed, by the way...)
